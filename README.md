# E-COMMERCE-SALES-ANALYSIS-DASHBOARD-EXCEL
Developed an interactive Ecommerce Sales Analysis dashboard to help stakeholders monitor sales performance, customer behavior, and revenue trends.

The organization relied on static reports that made it difficult to:
1.Track real time sales performance
2.Identify top-performing products
3.Analyze customer purchasing patterns
4.Compare regional sales growth
5.The goal was to design a solution that converts raw sales data into meaningful business insights.
6.Assisted in stakeholder discussions to capture reporting and analytics requirements.
7.Prepared Business Requirement Document (BRD) defining project scope, KPIs, and expected outcomes.
8.Created user stories with acceptance criteria for dashboard features (filters, revenue tracking, category analysis).
9.Defined functional requirements (sales trend analysis, product ranking, region-wise performance) and non-functional requirements (performance, usability).
10. business process flow for data extraction, transformation, and reporting.
11.Conducted gap analysis between existing reporting process and proposed automated dashboard solution.
12.Mapped business needs into system features such as dynamic filtering, KPI cards, and drill-down analysis.
13.Collaborated in an Agile environment to deliver features incrementally.
